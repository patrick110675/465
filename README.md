# 高峰競賽平台 V2.0.1 首頁管理版

英文：Peak Competition Platform

這版已可直接放到 GitHub Pages 或 Firebase Hosting。
目前前端使用 LocalStorage 示範運作；Firebase 設定檔已預留。

## 已完成

- 乾淨版 Dashboard
- 每日之星：只顯示今日加權第一名
- 每日報件：新增、查詢、彙總、Excel、PDF
- 排行榜：個人、區單位、隊伍、組別、職級、商品
- 管理中心：人員、商品、匯率、競賽、獎勵活動
- 匯入中心：人員、商品、匯率、業績、獎勵活動、競賽標準
- 外觀設定：背景、主色、文字、卡片、圓角、Banner
- 一鍵封存競賽
- Audit Log 雛形
- Firebase Hosting / Firestore Rules 骨架

## GitHub Pages 上線

GitHub → Settings → Pages → Deploy from branch → main → root → Save

## Firebase Hosting 上線

```bash
firebase login
firebase init hosting
firebase deploy
```

## 匯入欄位

人員：姓名、區單位、隊伍、組別、職級  
商品：商品名稱、商品代碼、年期、幣別、原始加權、競賽加權、商品類別  
匯率：年度、月份、美金匯率  
業績：日期、姓名、商品、保費  
獎勵活動：活動名稱、截止日期、計算方式、目標、獎金、指定商品  
競賽標準：競賽名稱、開始日期、結束日期、職級、加權目標、實收目標、獎勵

## V1.6 更新：匯率可修改
- 管理中心 → 匯率：新增「修改」按鈕。
- 修改年度、月份、美金匯率後按「儲存修改」。
- 新增匯率時若年度＋月份已存在，系統會直接更新該月份，不會重複新增。
- 每筆既有業績仍保留當時使用的匯率；修改匯率只影響之後新增或重新匯入的業績。

## V1.7 更新
- 人員管理加入搜尋：姓名、區單位、隊伍、組別、職級皆可搜尋。
- 人員管理加入快速匯入人員 Excel 按鈕。
- 人員管理加入統計摘要：人員數、區單位數、隊伍數、組別數。
- 人員 Excel 匯入支援更多欄位名稱：姓名、業務員、姓名(中文)、單位、隊名、小組、職稱。
- 匯入人員時會略過空白姓名，避免產生空資料。


## V1.8 更新：商品管理強化
- 商品管理加入搜尋：商品名稱、商品代碼、年期、幣別、類別、加權皆可搜尋。
- 商品管理加入幣別篩選：全部 / TWD / USD。
- 商品管理加入統計摘要：商品總數、台幣商品、美金商品、A&H 商品。
- 商品管理加入快速匯入商品 Excel 按鈕。
- 新增商品時若商品代碼已存在，系統會更新原商品，不會重複新增。
- 商品加權可輸入 300%、3、0.05 等格式，系統會自動轉換。
- 商品匯入支援更多欄位名稱：商品名稱、商品、商品名、商品代碼、險種、年期、繳費年期、原始倍率、競賽倍率、加權。
- 空白商品名稱會自動略過，避免產生錯誤資料。


## V2.0.1 更新：首頁管理
- 移除「重置示範資料」按鈕。
- 管理中心新增「首頁設定」。
- 首頁標準方塊可顯示或隱藏。
- 可自行新增與刪除首頁自訂方塊。
- 通訊處目標、已完成業績可手動輸入或使用自動計算。
- 新高峰／新極峰目標與已完成業績可手動輸入或使用自動計算。
- 我的獎勵活動可在管理中心新增、修改、刪除。
- 已填入 team-587a0 Firebase Web App 設定。注意：本版主要資料仍存於瀏覽器 LocalStorage，Firestore 集合式同步尚未完成。

## V2.0.2 獎勵活動進階版

新增：
- 獎勵活動可設定開始日與結束日。
- 可依競賽加權、實收、原始加權、A&H 或件數計算。
- 可指定商品大類、商品次類、多個商品、A&H、保障型。
- 可限制適用職級；資料結構亦預留區單位、隊伍與組別。
- 商品主檔新增商品次類、保障型、主約／附約欄位。
- 首頁獎勵卡會顯示期間、抓取條件、已完成、目標、達成率與還差多少。
- Excel 獎勵活動範本已同步擴充。

注意：此版本仍以瀏覽器資料儲存為主要運作方式；Firestore 分集合正式同步與批次初始化仍需後續完成。
